Overview

A CCPlayer plays a CCTrack by sending its CCSteps' CC messages over MIDI.

Generally you will not make instances of CCPlayer directly, and instead use #track_live_loop, which will create and manage a CCPlayer for you.

In the unlikely scenario that you want to manually drive a CCPlayer, see the PlayerBase documentation for details.

    The MIDI channel to use during playback. If nil, falls back to the global default set by Sonic Pi's use_midi_defaults, or to all channels (i.e. "*") if that was not set.

  • port (String, nil) (defaults to: nil) —

    The MIDI device to use. If nil, falls back in the same manner as channel.

  • debug (Boolean) (defaults to: false) —

    If true, the player will log detailed information about its state during playback.

def initialize(track, channel: nil, port: nil, debug: false)
  @channel = channel
  @port = port
  @midi_spi_kwargs = {}
  @midi_spi_kwargs[:channel] = channel unless channel.nil?
  @midi_spi_kwargs[:port] = port unless port.nil?

  super(track, debug: debug)
end
