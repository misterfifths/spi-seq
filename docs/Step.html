Overview

A Step represents a sequenceable MIDI note event which can appear in slots in the grid of a Track. See the documentation for StepBase for details about steps in general, and their probability and accumulation mechanics.

Accumulation on a Step applies a semitone offset to the #note by default, but can also target the #gate or #vel.

Note that Steps are immutable. The mutation methods provided here, like #with_note, return new Steps that have all the same attributes as the receiver, with just the described change.

Constructor Details

#initialize(note, vel: 127, gate: 1.0, prob: nil, accum_delta: 0, accum_max: 12, accum_min: 0, accum_mode: :wrap, accum_prob: nil, accum_target: nil) ⇒ Step

Constructs a Step.

Step.new is aliased to #S for convenience.

By default, accumulation on a Step manifests as a shift in the #note by the accumulated number of semitones. It can also target the #gate or #vel with the accum_target parameter.

You may find it more convenient to set the accumulation parameters after constructing a step with the StepBase#accum method.

Parameters:

  • note (MIDINote, String, Symbol, Integer) —

    The note this step will play when triggered (subject to accumulation). May be any value understood by MIDINote.new.

  • vel (Integer) (defaults to: 127) —

    The velocity to use when triggering a MIDI not event for the step. Values outside of 0 - 127 (inclusive) will be clamped to the nearest extreme. See #vel.

  • gate (Number) (defaults to: 1.0) —

    The gate for the note event, as a fraction of the duration of its slot. Values outside of 0.0 - 1.0 will be clamped to the nearest extreme. See #gate.

  • prob (Prob, Number, #call, nil) (defaults to: nil) —

    The probability that the step will trigger when its slot is played, or nil if it should always trigger. Non-Prob values will be converted as described.

  • accum_delta (Integer) (defaults to: 0) —

    The amount to add to the accumulation each time the step triggers during playback; see StepBase#accum_delta.

  • accum_max (Integer) (defaults to: 12) —

    Defines the maximum total accumulation; see StepBase#accum_max.

  • accum_min (Integer) (defaults to: 0) —

    Defines the minimum total accumulation; see StepBase#accum_min.

  • accum_mode (:freeze, :wrap, :reverse) (defaults to: :wrap) —

    Controls the behavior when the total accumulation exceeds the range defined by accum_min and accum_max. See StepBase#accum_mode.

  • accum_prob (Prob, Number, #call, nil) (defaults to: nil) —

    The probability that accumulation will trigger when the step does, or nil if it should always trigger. Non-Prob values will be converted to Probs as described for prob.

  • accum_target (Symbol, nil) (defaults to: nil) —

    The target for accumulation. If nil, defaults to :note for Steps and :value for CCSteps. :value is the only valid value for CCSteps. You may pass :note, :gate, or :vel for Steps, to direct accumulation to the corresponding attribute.

def initialize(note, vel: 127, gate: 1.0, prob: nil,
               accum_delta: 0, accum_max: 12, accum_min: 0,
               accum_mode: :wrap, accum_prob: nil, accum_target: nil)
  @note = MIDINote.new(note)

  @vel = vel.to_i
  if @vel < 0
    @vel = 0
  elsif @vel > 127
    @vel = 127
  end

  @gate = gate.to_f
  if @gate < 0.0
    @gate = 0.0
  elsif @gate > 1.0
    @gate = 1.0
  end

  super(prob: prob, accum_delta: accum_delta, accum_max: accum_max,
        accum_min: accum_min, accum_mode: accum_mode,
        accum_prob: accum_prob, accum_target: accum_target)
end

Instance Attribute Details

#gate ⇒ Number (readonly)

The gate for this step as a percentage 0.0 - 1.0. This is the percentage of the duration of a slot for which the step will trigger its note. For instance, a value of 0.5 means the note will last half the duration of the slot in the Track that contains it.

The note will not be played with a gate of 0. With a gate of 1, the note is "tied". That is, it will continue without interruption if there is a Step in the next played slot with the same note.
